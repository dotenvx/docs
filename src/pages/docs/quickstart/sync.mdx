## Create .env file

Create a `.env` file.

```bash {{ title: ".env" }}
# development
HELLO="World"
```

Next, set up it's `.env.vault`.

## Set up .env.vault

Run [dotenv-vault](https://github.com/dotenv-org/dotenv-vault)'s **new** command.

<CodeGroup>
```bash {{title: "npx"}}
npx dotenv-vault new
```

```bash {{title: "Homebrew"}}
dotenv-vault new
```
</CodeGroup>

<ScreenshotBrowser url='/assets/img/docs/dotenv-new.gif' www='npx dotenv-vault new' />

Running this command sets up your project on [Dotenv](https://dotenv.org) and creates your project's `.env.vault` file. It will look like this.

```ini {{ title: ".env.vault" }}
#/-------------------.env.vault---------------------/
#/         cloud-agnostic vaulting standard         /
#/--------------------------------------------------/
DOTENV_VAULT="vlt_1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a1a"
```

The `.env.vault` file uniquely identifies your project on Dotenv. You can think of it like a unique git url at GitHub. It identifies your project so that you and your teammates pull the correct `.env` file from Dotenv. [Read more about `.env.vault` files](/docs/security/env-vault).

Next, log in to your `.env.vault`.

## Log in .env.vault

Run the **login** command.

<CodeGroup>
```bash {{title: "npx"}}
npx dotenv-vault login
```

```bash {{title: "Homebrew"}}
dotenv-vault login
```
</CodeGroup>

<ScreenshotBrowser url='/assets/img/docs/dotenv-login.gif' www='npx dotenv-vault login' />

Running this command authenticates you against your project on [Dotenv](https://dotenv.org) and securely generates your `.env.me` credential. Your `.env.me` file will look like this.

```ini {{ title: ".env.me" }}
#/!!!!!!!!!!!!!!!!!!!!.env.me!!!!!!!!!!!!!!!!!!!!!!!/
#/ credential file. DO NOT commit to source control /
#/--------------------------------------------------/
DOTENV_ME="me_1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z1z"
```

The `.env.me` file uniquely authorizes you to access your project's env vault. You can think of it like your unique SSH key at GitHub. It identifies your machine so that you can safely and securely sync your `.env` file from Dotenv. [Read more about `.env.me` files](/docs/security/env-me).

Next, push your `.env` file.

## Push .env

Run the **push** command.

<CodeGroup>
```bash {{title: "npx"}}
npx dotenv-vault push
```

```bash {{title: "Homebrew"}}
dotenv-vault push
```
</CodeGroup>

<ScreenshotBrowser url='/assets/img/docs/dotenv-push.gif' www='npx dotenv-vault push' />

This securely pushes your `.env` file to your Dotenv Account. It works a lot like [git](https://git-scm.com/).

## Pull .env

You can pull the latest changes too. Run the **pull** command.

<CodeGroup>
```bash {{title: "npx"}}
npx dotenv-vault pull
```

```bash {{title: "Homebrew"}}
dotenv-vault pull
```
</CodeGroup>

<ScreenshotBrowser url='/assets/img/docs/dotenv-pull.gif' www='npx dotenv-vault pull' />
