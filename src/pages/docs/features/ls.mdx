# List .env files

## Installation

```bash
dotenvx ls
```

Lists out all .env* files in the folder and subfolders and prints them as a tree.

## Usage

Given you have a project monorepo like this:

```
apps/
  backend/
    .env
    .env.vault
  frontend/
    .env.production
    .env.vault
.env
```

It will output the following:

```bash
$ dotenvx ls
├─ .env
├─ frontend
│  ├─ .env.vault
│  └─ .env.production
└─ backend
   ├─ .env.vault
   └─ .env
```

### Filter by filename

Optionally, you can filter by specific filename(s):

```bash
$ dotenvx ls --env-file .env.vault
├─ frontend
│  └─ .env.vault
└─ backend
   └─ .env.vault
```

Or, use a wildcard:

```bash
$ dotenvx ls --env-file .env.prod*
└─ frontend
   └─ .env.production
```
