## Npm

`dotenvx` is made with javascript – so you can use it just like you use `dotenv` – as a drop-in replacement.

```bash
npm install @dotenvx/dotenvx --save
```
```js
// index.js
require('@dotenvx/dotenvx').config()

console.log(`Hello ${process.env.HELLO}`)
```

Find the [@dotenvx/dotenvx package here](https://www.npmjs.com/package/@dotenvx/dotenvx).

## Brew

Installing with [brew](https://brew.sh) is the most straight forward global install:

```bash
# install with Homebrew and then use the dotenvx command
brew install dotenvx/brew/dotenvx
```

Find the [dotenvx brew tap here](https://github.com/dotenvx/homebrew-brew).

## Shell

After brew our install.sh script is easiest. With curl:

```bash
curl -sfS https://dotenvx.sh | sh
```

Or with wget:

```bash
wget -qO- https://dotenvx.sh | sh
```

Read the [install.sh script here](https://github.com/dotenvx/dotenvx.sh/blob/main/install.sh).

## Docker

We also provide a [docker image](https://hub.docker.com/r/dotenv/dotenvx) for your convenience:

```bash
docker run -it --rm -v $(pwd):/app dotenv/dotenvx help
docker run -it --rm -v $(pwd):/app dotenv/dotenvx run -- node index.js
```

Or in any image:

```
FROM node:latest
RUN echo "HELLO=World" > .env && echo "console.log('Hello ' + process.env.HELLO)" > index.js
RUN curl -sfS https://dotenvx.sh/install.sh | sh
CMD ["dotenvx", "run", "--", "echo", "Hello $HELLO"]
```

## GitHub Releases

You can also download straight from [GitHub Releases](https://github.com/dotenvx/dotenvx/releases).

```bash
curl -L -o dotenvx.tar.gz "https://github.com/dotenvx/dotenvx/releases/latest/download/dotenvx-$(uname -s)-$(uname -m).tar.gz"
tar -xzf dotenvx.tar.gz
./dotenvx help
```

<Note>
  If you download manually you'll need to move the binary somewhere in your PATH.
</Note>

## Other

### Heroku Buildpack

For our Heroku friends, we have a [heroku buildpack](https://github.com/dotenvx/heroku-buildpack-dotenvx).

```bash
heroku buildpacks:add https://github.com/dotenvx/heroku-buildpack-dotenvx
```

The `dotenvx` binary will be available to your deployment so that you can use it in your Procfile like so.

```bash
# Procfile
web: dotenvx run -- node index.js
```

### Standalone

`dotenvx` is a standalone binary, so (if you want) you can just download it directly:

```sh
# download it to `./dotenvx`
curl -sfS --proto '=https' https://dotenvx.sh/$(uname)/$(uname -m).tgz | tar xz

# install it to `/usr/local/bin/dotenvx`
sudo install -m 755 dotenvx /usr/local/bin

# check it works
dotenvx help
```

Download a specific version:

```sh
# download version 0.6.9
curl -sfS --proto '=https' "https://dotenvx.sh/$(uname)/$(uname -m).tgz?version=v0.6.9" | tar xz

# check the version
./dotenvx --version
```

<Note>
  You can view available versions at [github.com/dotenvx/releases](https://github.com/dotenvx/releases/tree/main)
</Note>

### Npm local

Use dotenvx locally as a cli in your node project.

```sh
npm i @dotenvx/dotenvx --save
```
```json
{
  "scripts": {
    "start": "./node_modules/.bin/dotenvx run -- node index.js"
  },
  "dependencies": {
    "@dotenvx/dotenvx": "^0.6.0"
  }
}
```

### Npm global

You can also install globally using npm's `--global` flag.

```bash
npm install @dotenvx/dotenvx --global
dotenvx help
```

### Npx

Or prefer to run as needed, uses `npx`.

```bash
npx @dotenvx/dotenvx help
```

### pkgx

[pkgx](https://pkgx.sh) is a single, standalone binary that can run anything. Since it can run anything, it can run `dotenvx`:

```sh
pkgx npx @dotenvx/dotenvx help
```
